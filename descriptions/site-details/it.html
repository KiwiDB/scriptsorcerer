A front-end replacement for Jira Issue Tracker, written for Hobsons to meet the needs of a few different teams. Uses webservices provided by Jira to create a personalized experience. The system is built to meet the needs of these specific teams, but generalized to be usable by any team. This custom functionality includes team-specific issue analysis, read status based on last-update-versus-last-read time, and AJAX-based workflows. Also contains a wizard for issue creation to calculate due dates, assignee, and issue description content (using XML files to specify the details of those calculations).
