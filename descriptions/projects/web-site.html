The Bootstrap Build website is a tool I own and built as a supplement for webmasters who use Twitter Bootstrap elements in their own projects. Bootstrap provides for some great prebuilt elements, but customizing them for real-life use is a little more complicated.

The site uses a wizard-style interface to customize Bootstrap HTML based on the answers to relevant questions. Once an HTML block is generated, an on-site preview is generated along with the HTML itself, ready to be copied and used as desired. So far the site provides two generators, with more currently in development.

I am the sole contributor to this site -- webmaster, developer, designer, marketer, site security, etc. Most of the code associated with the site is available in my github account, altho the files with private or security information (mostly the PHP) is not tracked.

In 2013 I joined the Media Services team (a digital marketing team) at Hobsons, as developer. One of my major responsibilities on the team is the building and maintenance of marketing websites used by the greater team to generate contacts. These websites collect leads and submit them to different systems (the CRM this team uses internally, CRMs used by clients, different sales and marketing tools, along with emails and CSV exports). Before I took over responsibility for these sites, they were a mess. The process and technologies needed a full redesign, which I undertook:

  • Design Team. Together with the Media Services team leadership, we introduced a true design team to te mix. These were junior designers from elsewhere in the company (who, until this point, concentrated on print design). I worked with them to teach web design concepts. The designers were to be responsible for creating mockups, and starting the HTML build process before sending it and the site resources (images, fonts, etc) to me.
  • Website Backend. For years before I joined the process, our sites have been built using Joomla -- but only in the most academic sense. Few of the advantages a CMS provides were used in any way. I rethought and restructured the way our sites are built, introduced basic templates, modules and menus, along with more advanced concepts like custom menu item types and custom module types
  • Website Frontend. Our websites need to meet many requirements -- functional, fast, easily maintainable, maintainable by a team of individuals, representable in Joomla, and able to fit into the process of splitting design and build between two teams. I added two vital technologies to the sites: Twitter Bootstrap, which allows us to use standard elements, quick prototyping, and standardization for maintenance; and LESS CSS, making it easier to keep stylesheets organized and syntactically readable. We also pushed our sites to be fully responsive.
  • Submission Manager. The whole point of these sites is to generate leads through the on-site contact form. These leads need to be processed and submitted to any number of remote systems. The real-time processing code that was in place was error-prone, costly, and time consuming. I built a whole new system for dealing with submission, emphasising automation, self-correction and transparency. The new system uses an AWS SQS queue to store leads, and periodically submits them through to the appropriate endpoints by reading from configuration files. If a system goes down, the lead is left in the queue and submission is reattempted periodically until successful. All processes are logged and viewable by the administering team.
